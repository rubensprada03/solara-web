Cerveja Artesanal × Indústria 4.0

Automação, monitoramento remoto e controle de temperatura em processos de manufatura artesanal com ESP32, IoT e Google Apps Script — aplicados à produção real de uma Session Pilsen.

ESP32 · IoT Excel Session Pilsen Indústria 4.0 DS18B20 · Histerese Google Sheets API
⚗
01 — Resumo do Projeto

O que fizemos

Desenvolvemos, implementamos e avaliamos uma ferramenta de monitoramento baseada no Sistema em Chip ESP32 para automação e controle de temperatura em processos de manufatura artesanal, tomando como referência a produção de uma cerveja do estilo Session Pilsen.

O sistema integra o sensor digital DS18B20, um relé com acoplamento transistorizado para acionamento de um fogão elétrico e um algoritmo de controle on-off com histerese que estabilizou as variações críticas de mostura e fervura.

⚙ Hardware & Controle

O ESP32 lê continuamente o sensor DS18B20 e aplica lógica de histerese (±0,5°C) para acionar o relé de forma segura, protegendo o equipamento e mantendo o processo estável. Um transistor garante compatibilidade de tensão entre a placa e o relé mecânico.

☁ Nuvem & Dados

A cada 10 segundos, dados de telemetria são enviados via HTTP a uma API em Google Apps Script, que valida e insere as leituras em um Google Sheets. O histórico é então visualizado no Excel.

🌡 Resultados Alcançados

Temperatura média na mostura de 65,18°C (meta: 65°C), mash out em 76,12°C e taxa de entrega de dados de 98,4% de sucesso ao longo de toda a produção.

🍺 O Produto Final

Uma Session Pilsen de 5 litros produzida com 675g de Malte Pilsen e 75g de Malte Chateau Munich, lupulada com Nugget e Brewer's Gold, fermentada e maturada com controle automatizado de temperatura pelo ESP32.

02 — Referencial Teórico

Fundamentos

A base científica e tecnológica que sustenta o projeto une controle de sistemas, Internet das Coisas e a bioquímica do processo cervejeiro.

🔄

Controle On-Off com Histerese

O controle liga-desliga simples causa chaveamento rápido do relé perto do setpoint, reduzindo a vida útil do equipamento. A histerese cria uma margem de tolerância de ±0,5°C acima e abaixo do valor desejado, espaçando os acionamentos e garantindo estabilidade sem necessidade de controle PID.

📡

IoT e o ESP32

A Internet das Coisas conecta objetos físicos com sensores e software à internet para troca de dados. O ESP32 se destaca por alto desempenho, Wi-Fi integrado e baixo custo, sendo ideal para prototipagem de sistemas embarcados de monitoramento em tempo real com envio de telemetria.

🌾

Processo Cervejeiro — Fase Quente

A mostura (60–72°C) ativa enzimas que convertem amido em açúcares. O mash out a 76°C encerra a atividade enzimática. A fervura de 60 minutos esteriliza o mosto e permite a lupulagem com liberação de amargor e aroma. Cada faixa de temperatura define o perfil da cerveja final.

⚙
03 — Metodologia

Como construímos

A arquitetura do sistema integra hardware embarcado, lógica de controle e uma cadeia completa de processamento e visualização de dados em nuvem.

01

Hardware & Interface

Dois botões físicos permitem ajuste manual do setpoint. Display LCD exibe dados em tempo real. Um transistor garante compatibilidade de tensão entre o ESP32 (3,3V) e o relé (5V), protegendo a placa.

02

Software Embarcado (C++/Arduino)

Ciclo contínuo de leitura do DS18B20, verificação de botões e aplicação da histerese de ±0,5°C. Fogão liga quando temp cai abaixo do mínimo, desliga quando ultrapassa o máximo.

03

Transmissão & Armazenamento

A cada 10 segundos, dados são enviados via Wi-Fi usando requisições HTTP para a API em Google Apps Script, que valida e insere cada leitura em uma linha do Google Sheets com timestamp.

04

Visualização & Dashboard

Excel conectado diretamente ao Sheets transforma os dados em gráficos de linha e indicadores visuais em tempo real. Um site local no ESP32 permite controle imediato sem dependência de internet.

04 — Processo Produtivo

O Experimento

Validação prática durante a produção real de uma Session Pilsen de 5 litros, seguindo protocolo da empresa "Faz Tua Ceva" em quatro etapas cronometradas e monitoradas.

Etapa I

Preparação e Mostura (Mash)

3 litros de água aquecidos a 70°C, seguido da adição de 675g de Malte Pilsen e 75g de Malte Chateau Munich. O setpoint foi reajustado para 65°C após a queda de temperatura causada pelos grãos frios. O sistema manteve agitação manual por 60 minutos com controle automático do fogão.

60 min 65°C setpoint 675g Malte Pilsen 75g Chateau Munich
Etapa II

Clarificação e Mash Out

Temperatura elevada para 76°C por 15 minutos para encerrar a ação enzimática. Em seguida, lavagem com 4 litros de água a 76°C para extração dos açúcares residuais do bagaço, resultando em ~8 litros de mosto filtrado pronto para fervura.

15 min 76°C 4L água de lavagem ~8L mosto final
Etapa III

Fervura (Boil) e Lupulagem

60 minutos de fervura com adições cronometradas de lúpulo: 2,5g de Nugget no início para amargor base e 2,5g de Brewer's Gold nos últimos 10 minutos para sabor e aroma. Resfriamento imediato a menos de 35°C com resfriador de imersão.

60 min fervura 2,5g Nugget 2,5g Brewer's Gold <35°C pós-fervura
Etapa IV

Fermentação, Maturação e Envase

Fermento Ale adicionado ao mosto frio. O ESP32 controlou automaticamente as temperaturas: 18°C por 10 dias (fermentação ativa), 20°C por 4 dias (descanso de diacetil), 10°C por 7+ dias (maturação). Envase com 6,5g de açúcar por litro para carbonatação natural.

18°C — 10 dias 20°C — 4 dias 10°C — 7+ dias 6,5g açúcar/L

08 — Sistema de Alertas

Notificações por E-mail

Além do monitoramento visual no Excel, implementamos um sistema de alertas automáticos diretamente no Google Apps Script, acionado como trigger a cada nova leitura inserida na planilha.

A função validarTemperatura() lê o valor mais recente da planilha e classifica em três estados. O e-mail só é enviado quando há mudança de estado, evitando spam de notificações repetidas.

OK Temperatura entre 18°C e 24°C — processo normal, nenhum alerta enviado
18–24°C
⚠ Warning Temperatura fora do alvo mas dentro de ±5°C — e-mail de aviso enviado
<18 ou >24°C
🚨 Critical Desvio superior a 5°C do limite — e-mail de emergência enviado imediatamente
<13 ou >29°C
Como funciona o controle de estado O status anterior é persistido via PropertiesService do Apps Script. A cada execução, o status atual é comparado com o armazenado — e o e-mail só é disparado se houve transição (ex: OK → WARNING ou WARNING → CRITICAL), protegendo a caixa de entrada de mensagens repetidas.
Google Apps Script — validarTemperatura.gs 
function validarTemperatura() {

  const planilha = SpreadsheetApp
    .getActiveSpreadsheet()
    .getActiveSheet();
  const ultimaLinha = planilha.getLastRow();

  if (ultimaLinha < 2) return;

  const temperatura = parseFloat(
    planilha.getRange(ultimaLinha, 2)
      .getValue().toString()
      .replace(",", ".")
  );

  const min = 18, max = 24;

  const props = PropertiesService
    .getScriptProperties();
  let ultimoStatus = props.getProperty("status") || "OK";
  let statusAtual = "OK";

  if (temperatura < (min - 5) || temperatura > (max + 5)) {
    statusAtual = "CRITICAL";
  } else if (temperatura < min || temperatura > max) {
    statusAtual = "WARNING";
  }

  // Só envia se o estado mudou
  if (statusAtual !== ultimoStatus) {
    MailApp.sendEmail({
      to: "henriquemoreira1074@gmail.com",
      subject: `🚨 ALERTA: ${statusAtual}`,
      body: `Temperatura atual: ${temperatura}°C`
    });
    props.setProperty("status", statusAtual);
  }

}

O projeto reuniu de forma rica eletrônica, programação embarcada, redes de comunicação, desenvolvimento web e termodinâmica industrial, provando que a Indústria 4.0 é viável mesmo para processos artesanais de pequena escala.
